# 2️⃣ Capa de Dominio (entidades, VOs, enums, excepciones)

---

Recuerda que en esta capa de dominio es donde vive el corazón del negocio, cero dependencias externas, cero Spring,
cero JPA, solo Java puro.

Seguiremos el orden natural de construcción, de lo más simple a lo más complejo:

````
enums → Value Objects (VOs) → Excepciones → Entidades de Dominio 
````

Y lo haremos feature por feature siguiendo nuestro Vertical Slicing:

````
customer → account → transaction
````

---

## Customer

Enums — simples y directos, sin dependencias. Son el vocabulario del negocio.

````java
public enum CustomerStatus {
    ACTIVE,
    INACTIVE,
    BLOCKED
}
````

````java
public enum DocumentType {
    DNI,
    RUC,
    PASSPORT
}
````
